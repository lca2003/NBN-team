【二、完成的功能清单】
------------------------------------------------------------------------
[✅] 1. 单列信息流列表布局与滚动优化
       - 使用 RecyclerView + LinearLayoutManager 实现单列信息流。
       - 多 viewType 复用机制（cell 复用），满足课题“资源复用”要求。

[✅] 2. 广告卡片多样式（要求至少 3 种，已实现 3 种）
       - 大图卡：顶部大图占位 + 品牌/标题/摘要/标签/统计 + 互动栏。
       - 小图卡：左文右小图横向布局，信息密度更高。
       - 视频卡：封面 + 半透明遮罩 + 播放按钮 + “视频”角标，默认暂停态。
       - 每张卡左上角带“广告”角标，符合广告信息流场景。

[✅] 3. 顶部 Tab 切换与联动刷新
       - TabLayout 三个频道：精选 / 电商 / 本地。
       - 切换 Tab 即刷新对应频道数据；再次点击当前 Tab 回到顶部。

[✅] 4. 下拉刷新、上拉加载（含三态）
       - 下拉刷新：SwipeRefreshLayout，刷新后回到顶部。
       - 上拉加载更多：滚动接近底部自动预取下一页。
       - 加载中态：底部 footer 显示“加载中…”转圈。
       - 空态：频道无数据时居中提示，可点击重试。
       - 错误态：加载失败 footer 显示“加载失败，点击重试”，点击可重试。

[✅] 5. 广告详情页 UI 与返回位置保持
       - 独立 AdDetailActivity：返回栏 + 封面 + 标题/品牌/摘要 + 智能标签 + 统计 + 互动栏。
       - 视频详情：默认暂停，点击播放按钮切换“已暂停/播放中”。
       - 返回信息流时列表滚动位置自动保持（不重建列表）。

[✅] 6. 点赞 / 收藏 / 分享交互 UI 与动画
       - 点赞：心形图标灰→红切换，点亮时播放“放大回弹”彩蛋动画。
       - 收藏：图标灰→金切换。
       - 分享：弹出系统分享面板 + Toast 提示。
       - 页面切换动画：进入详情 slide_in_left，返回 slide_out_right。
       - 跨页状态同步：详情页点赞/收藏后，返回信息流自动同步（共享 InteractionStore）。

【三、关键技术决策（含原因，便于答辩陈述）】
------------------------------------------------------------------------
1. 多样式卡片用 RecyclerView 多 viewType
   - 按 AdContentType（大图/小图/视频）返回不同 viewType，复用对应 ViewHolder。
   - 三种卡片布局控件 id 统一命名，共用一个 AdViewHolder，差异仅视频卡多播放按钮，减少重复代码。

2. 标签 chip 用 TextView 自绘，不引入 Material Chip
   - 当前主题是 Theme.AppCompat，引入 Material Chip 需改 B/C 共用的 styles.xml，风险大。
   - 用 TextView + 圆角背景自绘 chip（TagChipBinder），零主题依赖，信息流与详情页共用。

3. 分页与上拉加载：UI 层垫片，不碰数据层
   - 现状：队友B 的 MockAdRepository 只有 3 条种子数据、无分页接口。
   - 方案对比：
       方案A：直接改 B 的 Repository 加分页 —— 越界，违反协作约定，否决。
       方案B（采用）：在 ui.feed 加分页垫片 AdCatalog，只通过 AdRepository 接口读种子数据，
                      客户端循环合成多页 + 模拟网络延迟/失败。
   - 收益：上拉加载、加载中/空态/错误态都能完整演示；等 B 出真实分页接口，只替换
           AdCatalog.loadPage 内部实现，UI 层零改动。

4. 跨页状态同步：单一数据源 InteractionStore
   - 课题要求详情页和信息流的点赞/收藏状态同步。
   - 在 ui.feed 建进程内单例 InteractionStore，按 adId 统一保存 InteractionState（复用 B 的模型）。
   - Feed 卡片与详情页读写同一份状态，避免各页复制孤立状态。

5. 列表位置保持
   - 详情页用独立 Activity，信息流 Fragment 留在返回栈，RecyclerView 位置由系统保留。
   - 返回时只在 onResume 刷新可见项互动状态，不重建列表，位置不丢。

6. 视频卡占位说明
   - AdItem 暂无 videoUrl（属 B 数据层），视频卡做状态化占位：默认遮罩+播放按钮（暂停），
     点击切换播放态文案。真实播放器接入点预留对接 C 的 VideoPlaybackManager。

【七、演示路径（自测要点）】
------------------------------------------------------------------------
1. 启动 App → “精选”频道单列信息流，含大图/小图/视频三种卡片。
2. 下拉 → 刷新转圈 → 回到顶部。
3. 持续上滑 → 底部“加载中…” → 追加卡片 → 到底显示“没有更多了”。
4. 切换“电商/本地”Tab → 列表联动刷新；无数据频道显示空态。
5. 点卡片 → 滑入详情页 → 点赞（变红+彩蛋）→ 返回 → 信息流点赞态已同步。
6. 收藏变金 / 分享弹出系统面板。
7. 视频卡详情页点播放 → “已暂停/播放中”切换。
